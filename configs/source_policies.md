# Source Policies

本文件定义 research-assistant 在网页与 skills 之间共享的默认来源策略。目标不是限制一切来源，而是优先保证：

- 来源可信
- 链接可追踪
- PDF 可落盘
- 结果能稳定复现

## 主来源优先级

默认优先使用以下来源：

1. `arXiv`
2. `OpenReview`
3. `ACL Anthology`
4. `CVF Open Access`
5. `PMLR`
6. `官方项目页 / 官方代码仓库`

## 补充来源

以下来源可作为补充，但不应在没有说明的情况下替代主来源：

- `Semantic Scholar`
- `Crossref`
- `Google Scholar`
- 会议官网索引页
- 作者个人主页

## 文献巡检规则

- 文献巡检榜单应优先来自用户指定的 `sources`。
- 如果为了补齐关键论文而使用了主来源之外的页面，需要明确标注为“补充项”。
- 不要把博客、二手解读或转载页当作论文主链接。

## PDF 下载规则

- 优先下载官方 PDF 或平台稳定 PDF：
  - `arXiv PDF`
  - `OpenReview PDF`
  - 会议官方 Open Access PDF
  - DOI 落地页解析出的官方 PDF
- 下载失败时，应返回候选页面或候选链接，而不是静默失败。
- 下载后应写来源 sidecar，记录：
  - 输入来源
  - 落地页
  - 解析出的 PDF 链接
  - 本地保存路径

## 开源链接策略

- 优先给官方仓库。
- 没有官方仓库时，再给高质量复现仓库。
- 没有公开代码时，明确写 `未发现公开代码`。

## 时效性规则

- 涉及最新论文、最新仓库状态、最新榜单或趋势判断时，要先核实再输出。
- 无法确认是否仍然有效时，要标注不确定性来源。
